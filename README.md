# Om Lathiya – Backend Engineer Portfolio

This repository contains the source code for my **personal backend engineering portfolio**, built as a **single-page React application** and deployed using **GitHub Pages**.

The portfolio is designed to clearly communicate **backend reliability, system design thinking, and business impact** to both non-technical founders and technical decision-makers.

🌐 **Live Site:** https://lathiyaom.github.io

---

## 🎯 Purpose

The goal of this portfolio is to:
- Explain backend value **within seconds**, not paragraphs
- Focus on **business outcomes**, not buzzwords
- Show depth **only when the reader wants it**
- Reduce cognitive load for non-technical clients

---

## 🧱 Structure (High-Level)

### 1. Immediate Clarity (Visible by Default)
- Clear value proposition
- Common backend failure risks explained in business terms

### 2. Trust & Proof
- Outcome-driven services
- Selected project highlights with measurable impact

### 3. Engineering Depth (Progressive Disclosure)
- Optional deep dives into:
  - Redis & caching strategies
  - Kafka & async systems
  - Concurrency & race conditions
  - Database design & optimization

---

## 🎨 Design Principles

- Calm and professional (no hype)
- Scannable in under 1 minute
- Progressive disclosure for technical depth
- Business-first messaging
- Clean, minimal UI

---

## 👥 Target Audience

- Non-technical founders
- Technical founders / CTOs
- Product managers evaluating backend reliability

---

## ⚙️ Tech Stack

- React 18
- Modern CSS (single-file, token-based)
- Semantic HTML
- Responsive, mobile-first layout
